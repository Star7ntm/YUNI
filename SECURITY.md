# 安全文档

## 🔒 安全措施

### 1. 身份验证与授权

#### JWT Token 认证
- **实现位置**: `backend/app/services/auth_service.py`
- **安全特性**:
  - 使用 HS256 算法签名
  - Token 包含过期时间（默认60分钟）
  - 生产环境必须设置 `YUNI_JWT_SECRET` 环境变量
  - 未设置时使用随机密钥（每次重启会失效，仅用于开发）

#### 密码安全
- **哈希算法**: Werkzeug PBKDF2 (默认方法)
- **密码强度要求**:
  - 至少6个字符
  - 必须包含小写字母
  - 必须包含大写字母
  - 必须包含数字

### 2. 文件上传安全

#### 文件类型验证
- **图片上传**: 仅允许 `.jpg`, `.jpeg`, `.png`, `.gif`, `.webp`
- **3D模型上传**: 仅允许 `.glb`, `.gltf`, `.obj`, `.fbx`, `.stl`, `.ply`, `.dae`, `.3ds`
- **音频上传**: 仅允许 `.wav`, `.mp3`, `.m4a`, `.ogg`, `.flac`

#### 文件大小限制
- 图片: 最大 5MB
- 3D模型: 最大 100MB
- 音频: 最大 50MB

#### 路径遍历防护
- 所有文件上传API都检查文件名是否包含 `..`, `/`, `\` 等路径遍历字符
- 使用UUID生成唯一文件名，避免文件名冲突和注入

#### 图片内容验证
- 使用PIL验证图片文件内容
- 自动转换和压缩图片
- 失败时自动删除已保存文件

### 3. 数据库安全

#### SQL注入防护
- 使用 SQLAlchemy ORM，所有查询都使用参数化
- 没有直接SQL拼接

#### 数据访问控制
- 所有用户数据操作都验证JWT token
- 用户只能访问自己的数据（通过uid验证）
- 模型删除时验证所有权

### 4. 输入验证

#### API输入验证
- 使用 Pydantic Schema 验证所有API输入
- 文件名、用户名等字符串都进行清理和验证
- 限制输入长度和格式

### 5. 敏感数据保护

#### 密码
- 密码从不以明文存储
- 密码哈希存储在数据库中
- 密码不在日志中输出

#### 用户信息
- `to_dict()` 方法不返回密码字段
- API响应中不包含敏感信息

## ⚠️ 已知安全风险

### 1. 验证码实现简化
**位置**: `backend/app/services/auth_service.py:validate_captcha()`

**问题**: 当前验证码验证过于简单，仅做基本格式检查。

**建议**: 
- 实现服务端token生成和存储（Redis/数据库）
- 添加过期时间验证（如5分钟）
- 添加防重放攻击机制
- 使用JWT签名验证

### 2. 头像上传依赖前端token
**位置**: `backend/app/api/v1/profile.py`

**当前状态**: 已修复，现在从JWT token获取用户，不再依赖前端传递username。

### 3. 历史记录无用户隔离
**位置**: `backend/app/api/v1/asr.py`

**问题**: 历史记录API未验证用户身份，所有用户共享历史记录。

**建议**: 
- 添加用户身份验证
- 在History模型中添加uid字段
- 按用户过滤历史记录

## 🔧 生产环境部署建议

### 1. 环境变量设置
```bash
# 必须设置
export YUNI_JWT_SECRET="your-strong-random-secret-key-here"

# 可选设置
export YUNI_HOST="0.0.0.0"  # 如果需要外部访问
export YUNI_PORT="8003"
export YUNI_DEBUG="False"   # 生产环境关闭调试
```

### 2. HTTPS配置
- 使用反向代理（Nginx/Apache）配置HTTPS
- 配置SSL证书
- 强制HTTPS重定向

### 3. CORS配置
- 限制允许的源域名
- 不要使用 `*` 通配符
- 在生产环境配置中设置 `cors_origins`

### 4. 文件存储
- 考虑使用对象存储（OSS/S3）而不是本地文件系统
- 配置文件访问权限
- 定期清理临时文件

### 5. 日志和监控
- 配置日志记录
- 监控异常请求
- 设置告警机制

### 6. 数据库安全
- 定期备份数据库
- 使用强密码（如果使用远程数据库）
- 限制数据库访问IP
